# Contributing

Thanks for contributing to iRetardgram.

## How to Contribute

1. Fork the repository.
2. Create a feature/fix branch from `main`.
3. Keep changes focused and small.
4. Test your changes before opening a PR.
5. Open a pull request with clear summary and rationale.

## Development Notes

- Keep shell scripts portable for Linux and macOS.
- Avoid broad find/replace operations without validation.
- Preserve patcher behavior unless change is intentional and documented.
- Update documentation when behavior changes.

## Commit Guidelines

- Use clear commit messages.
- Include what changed and why.
- Reference related issue IDs when available.

## Pull Request Checklist

- [ ] Scripts run without syntax errors
- [ ] README/docs updated if needed
- [ ] No unrelated formatting churn
- [ ] No secrets or keystores committed
